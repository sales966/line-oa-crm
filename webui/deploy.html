☁️ 阿里雲部署說明

僅「管理」角色可見

把原本只在區域網路跑的「LINE OA 客戶進度中樞」搬上阿里雲，讓同仁在任何地方都能用網域 + HTTPS 安全登入。全程約 30–40 分鐘。

⚠️ 核心安全原則（務必遵守）
  1. 對外只開 80/443，4680 埠絕不對公網開放（僅本機迴環 + 反代）。
  2. .env 必設 PUBLIC_MODE=1 — 登入頁不再把插件 Token 印給訪客，需登入後於後台取得。
  3. .env（含金鑰）、data/、storage/（客戶資料）永遠不進 git，只在伺服器上存在。

0. 事前準備

  • 一台阿里雲 ECS，作業系統選 Ubuntu 22.04 LTS（1 vCPU / 2GB 起步；檔案多建議 40GB+ 磁碟）。
  • 一個網域，例如 crm.example.com。
  • 本機現有資料：backend/data/app.db 與 backend/storage/。
  • 你的 OpenAI API Key 與現用的 EXTENSION_TOKEN（見本機 backend/.env）。

1. 阿里雲安全組（只開 80 / 443 / 22）

ECS 控制台 → 網路與安全 → 安全組 → 入方向，只保留：

協定埠授權對象用途
TCP22你的固定 IP（建議）SSH 管理
TCP800.0.0.0/0HTTP（導向 HTTPS + 憑證）
TCP4430.0.0.0/0HTTPS（同仁存取）

不要新增 4680 的規則。

2. 網域解析（DNS）

到網域 DNS 管理，新增一筆 A 記錄，指向你的 ECS 公網 IP：

主機記錄:crm
記錄類型:A
記錄值:  <你的 ECS 公網 IP>
TTL:     10 分鐘

等 ping crm.example.com 回你的 ECS IP 即生效（憑證申請前 DNS 必須先指好）。

3. SSH 登入並安裝 Docker

ssh root@<你的 ECS 公網 IP>

# 安裝 Docker（官方一鍵腳本）
curl -fsSL https://get.docker.com | sh

# 啟用並開機自啟
systemctl enable --now docker

# 驗證
docker --version
docker compose version

4. 取得程式碼

本公開庫只含程式碼，不含金鑰或客戶資料，可放心 clone：

mkdir -p /opt && cd /opt
git clone <你的公開庫 URL> lineoa
cd /opt/lineoa

5. 建立 .env（關鍵：PUBLIC_MODE=1）

cp backend/.env.example .env
nano .env

放在專案根目錄，至少要有：

# 公網加固 —— 務必設 1（登入頁不再顯示 Token）
PUBLIC_MODE=1

# 插件同步金鑰 —— 沿用本機同一串,否則現有插件會失聯
EXTENSION_TOKEN=<與本機相同的那一串>

# OpenAI 金鑰(留空則 AI 總結停用)
OPENAI_API_KEY=sk-...

LLM_MODEL=gpt-5.5
PORT=4680

# admin 初始密碼(僅 users 表為空時生效,登入後請立刻改)
ADMIN_INITIAL_PASSWORD=<自訂強密碼>

6. 啟動後端容器

cd /opt/lineoa
docker compose up -d --build

# 檢查(看到 "backend listening on http://0.0.0.0:4680" 即成功)
docker compose ps
docker compose logs -f backend
curl -s http://127.0.0.1:4680/api/health

7. 網域 + HTTPS（方案 A：Caddy，推薦最簡）

Caddy 會自動申請並續期 Let's Encrypt 憑證。

7-1 安裝 Caddy

apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | tee /etc/apt/sources.list.d/caddy-stable.list
apt update && apt install -y caddy

7-2 套用本專案附的 Caddyfile

# 把 Caddyfile 內的 crm.example.com 改成你的網域
nano /opt/lineoa/Caddyfile

cp /opt/lineoa/Caddyfile /etc/caddy/Caddyfile
mkdir -p /var/log/caddy
systemctl reload caddy
systemctl status caddy --no-pager

打開 https://crm.example.com 應見登入頁 + 有效憑證鎖頭。完成。

8. 遷移本機既有資料（對話 / 檔案）

8-1 本機：先停後端再打包

先關掉本機 start.bat（讓 SQLite WAL 落盤），在本機專案根用 PowerShell：

Compress-Archive -Path backend\data\app.db, backend\storage -DestinationPath lineoa-data.zip -Force

8-2 上傳到伺服器

scp lineoa-data.zip root@<ECS 公網 IP>:/opt/lineoa/

8-3 伺服器：停容器 → 解壓到 volume → 重起

cd /opt/lineoa
docker compose down

apt install -y unzip
unzip -o lineoa-data.zip -d _restore
mkdir -p data storage
cp _restore/backend/data/app.db data/app.db
cp -r _restore/backend/storage/* storage/
rm -rf _restore lineoa-data.zip

docker compose up -d
docker compose logs -f backend
帳號沿用資料庫內既有帳號；ADMIN_INITIAL_PASSWORD 只在 users 表為空時種子建立，遷移既有庫後不會覆蓋現有密碼。

9. 更新插件設定（改填 https 網域）

  1. 點插件圖示 →「開啟設定」。
  2. 後端網址改為 https://crm.example.com。
  3. API Token 維持 .env 內同一串（未變則不用動）。
  4. 儲存 → 到 chat.line.biz 重新整理 → 手動「立即同步」驗證。
公網模式下登入頁不再顯示 Token（改顯示「請登入後於設定取得」）。同仁需要 Token 時由管理員從 .env 提供。

10. 日常維運速查

cd /opt/lineoa

# 狀態 / 日誌
docker compose ps
docker compose logs -f backend

# 更新程式碼後重建
git pull
docker compose up -d --build

# 停 / 起 / 重啟
docker compose down
docker compose up -d
docker compose restart backend

每日自動備份（cron）

crontab -e
# 加一行:每天 03:00 打包 data + storage,保留最近 14 天
0 3 * * * mkdir -p /root/backups && tar czf /root/backups/lineoa-$(date +\%F).tar.gz -C /opt/lineoa data storage && find /root/backups -name 'lineoa-*.tar.gz' -mtime +14 -delete

疑難排解

症狀檢查
網域打不開 / 憑證失敗DNS A 記錄是否指向本機；安全組 80/443；systemctl status caddy
登入頁出現但 API 502docker compose ps 後端是否 Up；curl 127.0.0.1:4680/api/health
插件同步 401插件 Token 與 .env 的 EXTENSION_TOKEN 是否一致；網址是否已改 https
上傳大檔失敗反代體積上限是否 300MB（Caddyfile / nginx client_max_body_size）
遷移後資料沒出現app.db 是否放到 /opt/lineoa/data/app.db；是否 down 後才覆蓋再 up
AI 總結 503.env 的 OPENAI_API_KEY 是否填了；是否 docker compose up -d 重載 env
完整檔案版也在專案根 deploy-aliyun.md、Dockerfile、docker-compose.yml、Caddyfile。
